Focus Area: CTR & Engagement Priority Scoring

Dataset: March 2026 Daily Performance Data

Abstract

On large sites, spotting that bit of content that shows up but doesn't get the clicks is a struggle. This is where our machine learning pipeline for FlyRank comes in — highlighting the highest potential missed content opportunities.

Using compressed daily log data streamed through DuckDB, we processed more than 330k content items in March 2026 without hitting RAM limits. Once we stripped the features identified in early testing that caused target leakage, a final Random Forest model achieved a significant 0.8539 ROC-AUC score, substantially outperforming simple rule-based targeting and delivering a clean top 100 "playbook" ready for content teams to action.

1. Introduction and Problem Statement

Manual review of site content to ensure it's optimized for search is difficult and doesn’t scale when you’re trying to get work across a multitude of pages. High potential pages that receive relatively few impressions, yet a poor click-through rate when ranking reasonably well, present an obvious target. Since search engines are already indexing and presenting the content when querying for these pages, a click-through rate improvement on titles or meta descriptions offers a substantial and near-immediate improvement for the investment.

This project utilizes machine learning to build an automated system capable of selecting these pages from a vast corpus and prioritising them according to impact. It involves ingesting search query data, filtering only useful data features, generating predictions for which assets will benefit most, and returning that list to the user.

2. Data Overview

We used daily performance log streams streamed via Hugging Face Parquet logs:

  • Source Path: fact_content_daily_performance (partitioned by March 2026)
  • Data Timespan: March 1, 2026 through March 31, 2026
  • Total Content Items: 331,438
  • Data Privacy: All identifiers have been transformed into randomized, 16-character alphanumeric IDs for all data (content_hash_id). This ensures individual content remains protected.

3. Methodology

How We Identified High Impact Opportunity

A content opportunity is scored and identified (is_opportunity = 1) when a content article receives over 500 impressions but displays a CTR of less than 2.0% a piece:

CASE 
    WHEN AVG(gsc_clicks) / NULLIF(AVG(gsc_impressions), 0) < 0.02 
         AND AVG(gsc_impressions) > 500 THEN 1 
    ELSE 0 
END AS is_opportunity

Preventing Target Leakage

In a first attempt, the number of average impressions for a specific content ID was selected as a data feature. Because avg_impressions was so tightly correlated to the criteria that identify if a given content article was a hit (as our filter criteria is greater than 500 average impressions, yielding a 1.00 ROC-AUC achieved with this model), it would not differentiate how our content articles could be improved beyond fitting it for a target outcome and was consequently excluded from our training feature dataset.

Our finalized model now uses position-related features as data inputs $X$:

$$X = [\text{avg\_position}, \text{position\_volatility}]$$

Data Splitting Strategy

To isolate new, unseen data we used an 80% train and 20% test sample, maintaining balance for the positive class using stratified sampling.

4. Main Findings

After using our model to predict which articles will achieve the best balance of CTR with number of impressions, the model was run against a rule-based heuristic based off rank to determine a best practice comparison.

Model Architecture Features Used Validation ROC-AUC
Base Rule Heuristic (avg_position > 20) avg_position 0.5532
Random Forest Classifier avg_position, position_volatility 0.8539

As seen above, the Random Forest model outperformed the base rule, indicating it provided an approximate 54% improvement for rank quality. Interesting findings came from feature rankings: it appears that rank stability (position_volatility at over 50% of feature weighted impacts) has a significant impact.

5. Limitations and Realism

The system offers excellent decision support, but real-world implications apply:

  • High Rate of Class Imbalance: The conditions under which a content opportunity will achieve status mean our sample set is highly skewed, with less than 1% of our sample being an article of identified opportunity. This affects the recall of our model; without tuning the probabilities beyond these results, it will not recommend every article within the criteria of above a 2.0% click-through rate and above a few hundred impressions.
  • Directional Guidance: Our system guides towards potential improvement, not guaranteed results — search engine algorithms constantly evolve.
